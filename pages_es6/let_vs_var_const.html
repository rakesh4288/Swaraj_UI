Let vs Var vs Const

Back to Dashboard

Actually, both are using for creating a variable. Whereas Var keyword is the traditional approach and Let is the new keyword which is associated from the ES6

The Main difference is that Let keyword has a block level scope and Var is the function level scope.

Note: Function expression cannot hoisted which is created by Let Keyword

Note: Function expression cannot hoisted which is created by Var Keyword

More Details:
var: Declares variables with function or global scope and allows re-declaration and updates within the same scope.
Block Scope with let
This feature introduced in ES6. This means the variable is only accessible within the block (like loops or conditionals) where it is declared.


Immutability with const
const is used to declare variables that should not be reassigned after their initial assignment. This keyword is also block-scoped like let.
  • The variable country is declared with const, meaning its value cannot be reassigned after initialization.
  • The value of country (which is India) is printed to the console.
