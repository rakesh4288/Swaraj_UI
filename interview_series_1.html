forEach() .map() 
The forEach() method does not returns a  new array based on the given array. The map() method returns an entirely new array.
The forEach() method returns the " undefined". The map() method returns the newly created array according to the provided callback function.
The forEach() method doesn't return anything hence the method chaining technique cannot be applied here.  With the map() method, we can chain other methods like, reduce(),sort() etc.
It is not executed for empty elements. It does not change the original array.

Actually event.stopPropagation() stops an event from bubbling up the event chain,
whereas event.preventDefault() only prevent the browser's default action on that event from occurring, but the event still propogates up the event chain.

ES6 (ECMAScript 2015) and ES7 (ECMAScript 2016) are the standardized versions of JavaScript that introduced new features and improvements to the language.

ES6 is mainly known for its updates like - let, const, arrow functions, promises, and classes,
while ES7 focuses on fine-tuning and adding smaller improvements like the exponentiation operator and async/await.
